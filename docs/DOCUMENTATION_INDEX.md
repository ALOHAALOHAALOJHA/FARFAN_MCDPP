### Writing Guidelines
- **Audience-First**: Know who will read this

- **Code Examples**: Show, don't just tell
- **Diagrams**: Visualize complex concepts
- **Versioning**: Tag docs with system version

- **Critical Docs** (Operations, Runbook): Monthly review
- **Core Docs** (Architecture, Components): Quarterly review
